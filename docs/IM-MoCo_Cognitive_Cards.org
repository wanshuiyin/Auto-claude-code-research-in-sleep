* 论文说了什么

*问题*: MRI扫描时病人动了，图像就出现伪影。传统方法要么需要配对的干净/运动数据（临床里根本没有），要么算得太慢。怎么在只有运动数据的情况下自动修正？

*视角*: 把运动修正看作"内容"和"运动"的联合优化——用两个耦合的隐式神经网络（INR）分别表示，再通过MRI物理模型（k-space数据一致性）自监督训练。

*结果*: 实验证明比现有方法提升5% SSIM、5dB PSNR。在下游分类任务中，即使重度假影也能保持96%+准确率，而传统方法会掉到88%。

*一句话*: 这篇论文就是说——既然我们知道MRI是怎么从图像变到k-space的，那就让神经网络同时猜"原图长什么样"和"怎么动的"，用物理公式检查猜得对不对，不需要干净样本也能学。

* 对我意味着什么

*可能改变我的*: 对于逆问题，"测试时优化"可能比"训练一个大网络"更适合——特别是当我们有明确的物理 forward model 时。

* 技术要点速查

| 组件 | 关键参数 | 备注 |
|------|----------|------|
| Image INR | 3 layers, 256 ch, ReLU | 输出2通道复数图像 |
| Motion INR | 3 layers, 64 ch, Tanh | 输出变换网格 |
| kID-Net | U-Net, 4 levels, 16→128 ch | 预训练4200 epochs |
| Hash Encoding | tiny-cuda-nn | 多分辨率哈希网格 |
| DC Loss | k-space L2 | 数据一致性 |
| Entropy Reg | λ=1e-2 (annealed) | 梯度熵正则化 |
| Optimization | 200 iter, lr=1e-2 | 每样本测试时优化 |

* 复现信息

- 代码: https://github.com/multimodallearning/MICCAI24_IMMoCo
- 环境: Python 3.10, PyTorch, tiny-cuda-nn
- 数据集: NYU fastMRI T2 brain
- 训练时间: kID-Net ~4200 epochs; 推断 ~2-3 min/图像 (RTX 3090)
